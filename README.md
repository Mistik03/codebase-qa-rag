# codebase-qa-rag

A fully **local, privacy-preserving** Retrieval-Augmented Generation (RAG) system that
answers natural-language questions about a codebase — *"How does authentication work in
this project?"* — and cites the exact source files it used. Every model runs on your own
machine through [Ollama](https://ollama.com); **no source code ever leaves the device**.

This is the engineering artefact of a SEE University Capstone thesis:

> **A Local AI Assistant for Codebase Question-Answering: Comparing Retrieval Strategies
> for Angular and Python Projects.**
> Arb Xhelili — Mentor: Shqipe Salii — 2026.

## Research question

Which **code-chunking strategy** produces the best retrieval and answer quality for
codebase Q&A on consumer hardware? The system makes three axes swappable and measures
every combination:

| Axis | Options |
|------|---------|
| **Chunking** | `naive` (fixed token windows) · `component` (structure-aware: Angular component file-groups / Python modules) · `ast` (tree-sitter functions & classes) |
| **Retrieval** | `dense` (vector embeddings) · `hybrid` (dense + BM25, reciprocal-rank fusion) |
| **Codebase** | FormFit (Angular / TypeScript) · Microblog (Flask / Python) |

Answers are generated by **Qwen2.5-Coder 3B**, retrieval is scored with
precision/recall/MRR against hand-labelled gold files, and answer quality is graded by a
**Llama-3.1-8B** LLM judge that is itself validated against manual grading.

## Architecture

```
codebase  ─►  loaders  ─►  chunker  ─►  embeddings (nomic-embed-text)  ─►  ChromaDB
                                                                              │
question  ─────────────────────────────────────────────►  retriever  ◄───────┘
                                              (dense | hybrid + BM25, RRF)
                                                          │  top-k chunks
                                                          ▼
                                     Qwen2.5-Coder 3B  ─►  answer + file citations
```

## Quick start

```bash
# 1. Prerequisites: Python 3.11+, Ollama, and the models
ollama pull qwen2.5-coder:3b
ollama pull nomic-embed-text
ollama pull llama3.1:8b

# 2. Install
python -m pip install -r requirements.txt

# 3. (Python benchmark) clone the open-source codebase under test
git clone https://github.com/miguelgrinberg/microblog external/microblog

# 4. Ask a question
python -m experiments.ask --codebase microblog --strategy ast --method hybrid \
    "How does user authentication work?"

# 5. Run the full experiment grid and analyse
python -m experiments.run_grid
python -m experiments.analyze

# 6. Serve the web UI
uvicorn api.main:app --reload
```

See [`docs/SetupGuide.pdf`](docs/SetupGuide.pdf) for a complete walkthrough.
